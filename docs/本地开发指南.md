## 🛠️ 本地环境准备

### 1. 安装必要软件

#### Windows/macOS/Linux 通用：
- **Git**：版本控制工具
  - 下载地址：https://git-scm.com/downloads
  - 安装后验证：打开终端输入 `git --version`

- **Node.js**：JavaScript运行环境（可选，用于运行一些自动化脚本）
  - 下载地址：https://nodejs.org/
  - 推荐安装LTS版本
  - 安装后验证：打开终端输入 `node --version`

- **文本编辑器**：推荐使用 VS Code
  - 下载地址：https://code.visualstudio.com/

### 2. 克隆项目到本地

打开终端（Command Prompt/Terminal），执行以下命令：

```bash
# 克隆仓库
git clone https://github.com/landuo1999/Zhouguan-study.git

# 进入项目目录
cd Zhouguan-study
```

### 3. 检查Git分支

⚠️ **重要提醒**：项目有两个分支，GitHub Pages 从 `main` 分支构建！

```bash
# 查看当前分支
git branch

# 如果不在 main 分支，切换到 main
git checkout main
```

## 🎯 日常操作指南

### 场景1：添加新文章

#### 步骤1：准备文章信息
每篇文章需要以下信息：
```json
{
  "id": "唯一标识符（如：20240101）",
  "title": "文章标题",
  "url": "微信公众号文章链接",
  "category": "分类标签",
  "date": "发布日期（如：2024-01-01）",
  "description": "文章简介（可选）"
}
```

**分类标签可选值**：
- `cad` - CAD技术教程
- `su` - SU软件
- `adobe` - Adobe软件(PS/ID)
- `ppt` - PPT技巧
- `ai` - AI和效率工具
- `system` - 电脑系统和软件
- `gzh` - 公众号运营
- `life` - 个人/娱乐类

#### 步骤2：更新文章数据库
用文本编辑器打开 `articles-database.json`，在数组末尾添加新文章对象。

#### 步骤3：更新网站
有两种方式：

**方式A：手动更新（推荐初学者）**
1. 打开 `index.html`
2. 找到文章列表部分
3. 手动添加新文章的HTML代码

**方式B：使用自动化脚本（需要Node.js）**
项目中有一些辅助脚本可以帮助更新网站（如 `generate-index-from-db.js`）。

#### 步骤4：备份网站文件
更新完成后，复制 `index.html` 到 `website/` 文件夹作为备份。

## 🚀 部署到GitHub

### 标准工作流程

每次修改后，按照以下步骤提交更改：

```bash
# 1. 查看修改的文件
git status

# 2. 添加修改的文件
git add index.html
git add articles-database.json
# 如果修改了其他文件，也要添加

# 3. 提交更改
git commit -m "描述你的修改，比如：添加3篇新CAD文章"

# 4. 确保在 main 分支
git checkout main

# 5. 推送到GitHub
git push origin main
```

### 如果遇到冲突

```bash
# 先拉取远程更改
git pull --rebase

# 解决冲突后
git add .
git rebase --continue
git push origin main
```

## ⚠️ 注意事项和常见问题

### 1. 分支问题（最重要！）
- ❌ 不要在 `master` 分支上工作
- ✅ 始终在 `main` 分支上工作
- GitHub Pages 只从 `main` 分支构建网站

### 2. 推送到GitHub后网站没更新？
- 等待5-10分钟，GitHub Pages需要时间部署
- 确认你推送到了 `main` 分支
- 检查GitHub仓库的 Settings → Pages 页面

### 3. 如何恢复到之前的版本？
```bash
# 查看提交历史
git log

# 恢复到某个版本
git reset --hard 版本号
```

### 4. 图片和音乐文件
- 图片放在 `images/` 文件夹
- 音乐放在 `music/` 文件夹
- 引用时使用相对路径，如 `images/xxx.jpg`
