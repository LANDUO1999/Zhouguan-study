## 🛠️ 管理工具使用指南

### 1. manager.html - 文章管理工具
**用途**：添加新文章  
**使用流程**：
1. 先在"生成网站"标签页加载 `articles-database.json`
2. 在"添加文章"标签页填写文章信息
3. 可以选择子分类（自动从现有数据提取）
4. 下载更新后的数据库
5. 手动更新 `index.html`

### 2. category-manager.html - 分类管理工具
**用途**：查看和修改文章分类  
**使用流程**：
1. 加载 `articles-database.json`
2. 查看所有分类和文章
3. 重命名分类
4. 批量移动文章分类
5. 下载更新后的数据库

### 3. music-manager.html - 音乐管理工具
**用途**：添加和管理音乐  
**使用流程**：
1. 选择音乐文件，自动预览
2. 填写歌曲信息（歌名、艺术家、emoji封面）
3. 添加到播放列表
4. 可以调整顺序、删除歌曲
5. 下载 `playlist.json`
6. 生成 `index.html` 的播放列表代码
7. 手动更新 `index.html`

## 📋 标准工作流程

### 添加新文章
1. 用 `manager.html` 加载数据库
2. 添加文章信息
3. 下载 `articles-database.json`
4. 手动在 `index.html` 中添加文章HTML
5. 复制 `index.html` 到 `website/` 备份
6. 提交到 GitHub：
   ```bash
   git add index.html articles-database.json
   git commit -m "添加新文章"
   git push origin main
   ```

### 修改文章分类
1. 用 `category-manager.html` 加载数据库
2. 修改分类或移动文章
3. 下载更新后的数据库
4. 手动更新 `index.html` 中的分类显示
5. 提交到 GitHub

### 添加新音乐
1. 用 `music-manager.html` 添加音乐
2. 把音乐文件复制到 `music/` 文件夹
3. 下载 `playlist.json`
4. 生成并更新 `index.html` 中的播放列表
5. 提交到 GitHub：
   ```bash
   git add index.html music/
   git commit -m "添加新音乐"
   git push origin main
   ```

## ⚠️ 重要提醒（必须记住！）

### Git分支
- ❌ **不要用 master 分支**
- ✅ **必须用 main 分支**
- GitHub Pages 只从 main 分支构建

### 部署延迟
- 推送到 GitHub 后，等待 **5-10分钟** 网站才会更新
- 不要急，多等一会儿

### 文件备份
- 修改前先备份重要文件
- `website/index.html` 是备份位置
- 定期下载整个项目到本地

### 先加载数据库
- 使用管理工具时，**先加载数据库文件**
- 这样才能获取现有的子分类等信息

## 🔧 常见问题解决

### 网站没更新？
1. 确认推送到了 main 分支
2. 等待 5-10 分钟
3. 检查 GitHub 仓库的 Settings → Pages

### Git 冲突？
```bash
git pull --rebase
# 解决冲突后
git add .
git rebase --continue
git push origin main
```

### 想恢复旧版本？
```bash
git log  # 查看历史
git reset --hard <版本号>
```
